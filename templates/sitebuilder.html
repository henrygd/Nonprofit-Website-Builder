Donations

We will use Stripe as the donation processor. Stripe has no history of shady behavior, no monthly fees, and the percentage cost per transaction is very reasonable. They are also developer friendly, which means we can allow users to pay through a slick, straightforward, custom interface without ever leaving the site.

You will have to enter your website address when signing up. If you don't want to use a .com or .org domain ($10 to $12/year), sign at GitHub.com using your organization name as your username, then enter https://you.github.io on Stripe- replacing you with your GitHub username.

If you do want to buy a .com or .org (I strongly recommend doing so), Namecheap is the best option, and is what I use. Click the link, search for your domain, buy it, and use your new address for the Stripe sign up. Setting up your new domain through Namecheap is easy and I will walk you through it later!

After you've signed up on Stripe, copy your publishable API key (not the test keys or the secret key) and paste in the field below. Make sure it is the LIVE PUBLISHABLE KEY (matches the format below) or donations WILL NOT WORK.

Lastly, go to the server page I set up for these donations, enter and verify your publishable and secret LIVE API keys. The live keys, like the one you entered above -- NOT the test keys. Don't worry about the other instructions on the page; everything will be set up automatically.

Initial set up instructions

You must first have a Google Calendar API Key:

  1. Go to the Google Developer Console and create a new project (it might take a second).
  2. Once in the project, go to APIs & auth > APIs on the sidebar.
  3. Find "Calendar API" in the list and turn it ON.
  4. On the sidebar, click APIs & auth > Credentials.
  5. In the "Public API access" section, click "Create new Key".
  6. Choose "Browser key".
  7. If you know what domains will host your calendar, enter them into the box. Otherwise, leave it blank. You can always change it later.
  8. Your new API key will appear. Copy and paste it in the top field to the right, then reopen this window to complete the instructions.

Make your Google Calendar public:

  1. In the Google Calendar interface, locate the "My calendars" area on the left.
  2. Hover over the calendar you need and click the downward arrow (or create a new calendar with the settings below).
  3. A menu will appear. Click "Share this Calendar".
  4. Check "Make this calendar public".
  5. Make sure "Share only my free/busy information" is unchecked.
  6. Click "Save".

Obtain your Google Calendar's ID:

  1. In the Google Calendar interface, locate the "My calendars" area on the left.
  2. Hover over the calendar you need and click the downward arrow.
  3. A menu will appear. Click "Calendar settings".
  4. In the "Calendar Address" section of the screen, you will see your Calendar ID. It will look something like "randomlettersandnumbers@group.calendar.google.com".
  5. Close this window and paste your ID in the remaining field to the right.

Hosting free through GitHub

If you haven't already, sign up at GitHub (free) using your organization name as your username. Check your email and verify your account -- this is required to publish your site.

In the upper-right corner of any page, click +, and then click "New repository."

Name your repository you.github.io, replacing you with your username as shown in the example below. Check "Initialize this repository with a README." Adding a description is optional.

On the next page, click the + shown below to add a new file.

Name your file index.html (see below). Now find the index.html file you downloaded here earlier, right click on it and open in a plain text editor (such as Notepad on Windows -- please don't use Word). Alternatively, you may open in your browser and press Ctrl + U to view page source. Copy all the text (on Windows you can use Ctrl + A to select all, Ctrl + C to copy) and paste everything in the large text area on GitHub. Finally, click the button at the bottom of the page (not shown in the image) to commit the file.

Congratulations!

It may take up to an hour to publish (in the meantime a GitHub 404 page will show, this is normal), but your site will soon be online at yourusername.github.io!

While waiting, I'd greatly appreciate it if you'd star this tool on GitHub at this page.

If you've waited over an hour, have verified your address, and your site still doesn't show, email me at hank@henrygd.me. I will do what I can to help.

Cloudinary set up

Finding and optimizing your image

I highly recommend using either a photo you produced yourself or a photo that you know for sure is royalty free. Two of my favorite places for beautiful royalty free images are PEXELS and Unsplash. Check this list for other great sites.

After you find your image, you should optimize it for web use. This is a quick process and can drastically reduce load times. Go to this site or this site, upload your image, set the width to 1800, and submit. Download the resulting image. This is what you will upload to Cloudinary.

Cloudinary

It's free so sign up here. After that, click "Media Library" in the left panel and upload your image by dropping it on the square or browsing your computer. Once you see it below in your library, click it and copy the URL shown directly under the image (see example below). Paste that URL into the field in the panel to the right and click "Test Image."

Mandrill set up

When you reach the screen below, click "Get API Keys."

On the next screen, click "+Add API Key".

Edit your key (can be done at any time on the settings screen), click "Only Allow This Key To Use Certain API Calls," and check ONLY the "Send" option.

Save your key, copy it, and paste in the field in the sidebar to the right.

Note: One drawback to free serverless hosting on GitHub is that your Mandrill API key will be visible to others. In theory, a malicious user can find your key and use it to send their own emails. As long as you don't enter your credit card information, the worst outcome is that your 12,000 free monthly emails will be used up and your contact form will, for the rest of the month, display an error message asking the user to email directly. I've never encountered this issue, but keep in mind that it is possible. You may set up custom alerts (Account>Alerts) to notify you if your usage is higher than normal.
